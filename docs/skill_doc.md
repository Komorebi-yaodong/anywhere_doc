# Skill 技能库 📚(SKILL) 使用说明文档

**文档更新时间：2026年4月16日**

Anywhere Desktop 引入了对 **Skill** 的深度支持，目的是让 AI 学会一整套可复用的工作流程。如果说 MCP 是 AI 的“手”，那 Skill 就是 AI 的“方法论”与“SOP”。

通过 Skill，您可以把代码审查规范、日报模板、写作流程、运维手册等内容封装为可重复调用的技能包，在桌面端对话中按需挂载与执行。

---

## 1. 核心原理

### 1.1 渐进式披露 (Progressive Disclosure)

Anywhere Desktop 的 Skill 设计遵循渐进式披露原则：

* **启用阶段**：AI 先只知道技能的名称和简介；
* **执行阶段**：当 AI 判断当前任务需要某个 Skill 时，系统再把该 Skill 的完整指令和相关附件注入上下文。

这样既节省 Token，也避免把大量 SOP 文本长期塞入上下文窗口。

### 1.2 本地化存储

Skill 是完全本地化的：

* **存储位置**：在主控台 Skill 页面中配置技能目录；
* **不同步原则**：由于不同电脑环境、路径和依赖链不同，Skill 不建议直接随 WebDAV 同步。更推荐通过导出 / 导入的方式迁移。

---

## 2. 目录结构规范 (重要)

Skill 基于文件系统组织，建议按如下结构维护：

```text
D:\skills\
│
├── git-commit\
│   ├── SKILL.md
│   ├── styles.css
│   ├── rules.json
│   └── scripts\
│
└── code-review\
    └── SKILL.md
```

### 注意事项

1. 在桌面端中应选择 **包含多个 Skill 文件夹的父目录**；
2. 每个 Skill 目录下必须包含入口文件 `SKILL.md`；
3. 其它附件文件可在执行时由 AI 读取或引用。

---

## 3. 创建与编辑 Skill

### 3.1 界面操作

在 Skill 页面中点击 **“+ 新增技能”**，桌面端会提供可视化编辑器：

* **基本信息**：名称、描述；
* **详细指令**：即 Skill 的 SOP；
* **文件管理**：可附带模板、配置文件、示例代码、规范文档等。

### 3.2 高级配置：SKILL.md

如果您喜欢直接写文件，也可以手工维护 `SKILL.md`。它通常使用 YAML Frontmatter：

```markdown
---
name: Git 提交助手
description: 生成标准提交信息
disable-model-invocation: false
context: fork
allowed-tools: [read_file]
---

# 这里开始写详细指令
你是一个 Git 专家，请根据代码变更生成符合 Conventional Commits 的提交信息。
```

### 3.3 导入导出

桌面端支持通过文件夹、`.skill` 包或 `.zip` 包导入 Skill，也支持导出当前 Skill 以便迁移到其他电脑。

## 4. 运行模式

### 🔹 普通模式 (Direct Mode)

* **机制**：直接把 Skill 指令注入当前对话，由当前 AI 执行；
* **适用**：轻量级流程、格式化任务、简单 SOP 场景；
* **优点**：快、上下文连贯。

### 🔸 子智能体模式 (Sub-Agent / Fork)

* **机制**：基于桌面端内置的 Super-Agent，在后台拉起子智能体专门执行该 Skill；
* **适用**：复杂任务、长链路任务、多步骤工具调用任务；
* **优点**：
  * 中间步骤不污染主对话；
  * 子智能体更专注；
  * 可限制允许使用的工具范围。

## 5. 常见问题 (FAQ)

**Q1: 为什么启用了 Skill，AI 却说不会做？**

> **A:** 请检查：
>
> 1. Skill 是否启用；
> 2. 当前对话是否已勾选该 Skill；
> 3. 相关 MCP 工具是否可用。很多 Skill 本质上依赖 MCP 执行落地动作。

**Q2: Skill 可以用变量吗？**

> **A:** 可以。桌面端可支持变量替换，例如参数注入、唯一会话 ID 等，具体以 Skill 语法和运行链路为准。

**Q3: 如何让 AI 自动执行 Skill？**

> **A:**
>
> 1. 可在快捷助手中把某个 Skill 设为默认挂载；
> 2. 也可在定时任务中覆盖挂载 Skill；
> 3. 对于特定角色型助手，这样可以实现“打开即具备一套专业工作方法”。

**Q4: 子智能体执行卡住怎么办？**

> **A:** 通常是因为任务过于复杂、工具链受限、网络异常或步数预算不足。可改写 Skill、放宽工具权限，或拆分任务。

**Q5: 全局追问可以发给正在执行 Skill 的窗口吗？**

> **A:** 可以。只要独立窗口仍存在，您就可以继续追加文本、图片或文件给它，让当前上下文获得补充信息。

**Q6: 为什么重装系统后 Skill 页面空了？**

> **A:** 大多是因为本地 Skill 路径配置丢失或设备环境变化导致。重新在桌面端中选择原来的 Skill 目录即可恢复。
