# 定时任务 ⏰(Scheduled Tasks) 使用说明文档

**文档更新时间：2026年4月16日**

**定时任务** 是 Anywhere Desktop 的核心自动化能力之一。它允许您把 AI 工作流交给桌面端后台长期运行，让 AI 在本机环境中定时执行信息搜集、文件整理、报表生成与系统运维类任务。

您可以设置任务让 AI 每天早上整理新闻、每小时检查一次本机或服务器状态，甚至让 AI 借助内置 MCP 工具自行创建、修改和管理任务队列。

---

## 1. 快速开始

在 Anywhere Desktop 主控台中，点击顶部的 **“定时任务”** 标签页即可进入任务管理界面。

![定时任务界面](..\image\定时任务界面.png)

### 1.1 创建任务

1. 点击左侧列表底部的 **“新建定时任务”** 按钮。
2. 输入任务名称（例如：`每日早报`），回车确认。
3. 在右侧详情区配置触发规则、执行对象和指令内容。
4. 打开右上角开关，启用该任务。

### 1.2 核心概念

* **触发规则 (Trigger Rules)**：决定任务何时在桌面端后台自动运行。
* **目标助手 (Target Agent)**：决定由哪个 AI 助手执行任务。
  * *注意*：通常只有 **独立窗口** 模式的快捷助手适合作为任务执行对象，因为它支持完整上下文、多轮会话和工具调用。
* **指令内容 (Prompt Content)**：发送给 AI 的具体任务说明。
  * *建议*：任务属于无人值守场景，指令应尽量自洽、明确、可执行。例如：“请调用 web_search 搜索今日 AI 资讯，整理成 Markdown 并使用 write_file 写入桌面的 news.md 文件。”

---

## 2. 触发规则详解

Anywhere Desktop 提供了灵活的调度策略，可覆盖绝大多数桌面自动化需求：

* **单次执行 (Once)**：在指定日期与时间执行一次，随后自动停用。
* **间隔触发 (Interval)**：每隔 X 分钟执行一次。
  * *每日起始时间*：设置当天的对齐起点。
  * *生效时间段*：可配置一个或多个有效执行时间窗，例如 `09:00 - 12:00`、`14:00 - 18:00`。
* **每日定时 (Daily)**：每天固定时间执行。
* **每周定时 (Weekly)**：在每周指定日期与时间执行。
* **每月定时 (Monthly)**：在每月特定日期执行。

这些策略可用于新闻收集、备忘提醒、日志归档、系统巡检、日报周报生成等场景。

## 3. 高级配置

### 3.1 默认助手 (Default Agent)

如果您不想专门创建一个独立快捷助手，也可以使用系统内置的默认助手执行任务：

* 点击任务页中的相关设置入口，为默认助手指定模型。
* 默认助手不带额外角色预设，更适合执行通用、结构化的自动化命令。

### 3.2 临时扩展能力 (Overrides)

您可以为特定任务临时挂载额外能力，而不必修改助手本身的全局配置：

* **覆盖 MCP 服务**：任务执行时临时可用的工具集合。
* **覆盖 Skill 技能**：任务执行时临时启用的 SOP / 知识规范。

这使得同一个助手可以在不同任务里承担完全不同的职责。

### 3.3 后台无人值守行为

为了不打断您正在进行的工作，桌面端对任务执行做了不少收口：

* **独立窗口执行**：任务通常在独立对话窗口中运行，便于后续追踪执行上下文。
* **自动保存会话**：开启后，任务执行过程会自动静默保存到本地对话目录。
  * *命名格式*：`[任务名]-[时间戳].json`
* **执行完自动关闭**：适合“跑完即走”的无人值守任务。

---

## 4. 历史记录与监控

在任务详情页底部，您可以查看最近执行记录：

* **状态标签**：显示某次执行是成功、失败还是超时。
* **日志追溯**：若开启自动保存，成功执行通常会生成可恢复的会话文件。
* **立即运行测试**：强烈建议在正式启用前先手动运行一次，验证 Prompt、模型与 MCP 权限是否正确。

## 5. 进阶：让 AI 自我管理任务

得益于内置 **Task Manager** MCP，您可以让一个“秘书型”助手帮您直接管理任务系统。

**示例：**

> **用户**：帮我安排一个任务，从下周一开始，工作日每天早上 9 点自动搜索 AI 前沿资讯，总结成中文并写到桌面的 `news.md` 里。

AI 可以在后台：

1. 分析需求并调用创建任务工具；
2. 自动填写调度参数、执行时间、执行对象与任务指令；
3. 在桌面端中直接生成并启用该任务。

这也是 Anywhere Desktop 从“聊天工具”走向“本地 AI 自动化工作站”的关键能力。

## 6. 常见问题 (FAQ)

**Q1: 电脑休眠或退出应用后，任务还会执行吗？**

> **A:** 不会。Anywhere Desktop 的任务调度依赖本机运行中的桌面应用。如果电脑休眠、关机，或应用完全退出，任务不会继续执行。

**Q2: 为什么任务触发了，但窗口空白或一直转圈？**

> **A:**
>
> 1. 检查网络连接是否正常；
> 2. 检查目标助手或默认助手是否已配置有效模型与 API Key；
> 3. 检查任务指令是否过于复杂，建议先点击“立即运行”做人工回归测试。

**Q3: 多个任务同一时间触发会怎样？**

> **A:** 桌面端架构支持多窗口并发执行，但会同时消耗网络、模型额度与系统资源。建议尽量错开高频重负载任务。

**Q4: 为什么我清空了任务历史，但本地 JSON 还在？**

> **A:** 历史面板与磁盘文件是两层概念。若文件仍被占用、仍在同步，或清理时未勾选联动删除，本地 JSON 可能会保留，届时可手动前往目录处理。
