# 快捷助手 🚀(Quick Prompts/Agents) 使用说明文档

**文档更新时间：2026年4月16日**

**快捷助手** 是 Anywhere Desktop 的核心能力之一。它不是简单的“提示词模板”，而是可以被视为一组可复用、可配置、可直接召唤的桌面 AI 工作单元。您可以为每个助手配置独立的模型、系统提示词、触发方式、窗口外观、MCP 工具与 Skill 技能。

在桌面端中，快捷助手既可以通过主控台创建和管理，也可以通过全局快捷键、快捷召唤窗口（Quick）、快捷输入（Fast Input）以及全局追问能力，在系统任意位置快速投递内容与执行任务。

---

## 1. 核心模式说明

在创建快捷助手前，请先理解两种主要显示模式：

### 🚀 快捷输入 (Fast Input)

* **场景**：适合“阅后即焚”的一次性任务，例如划词翻译、代码解释、文案润色、命名建议。
* **交互**：调用后，会出现一个轻量级悬浮输入条。AI 完成输出后，可直接复制 / 粘贴结果，窗口随任务结束快速关闭。
* **特点**：启动快、干扰少、适合高频轻任务。
* **限制**：不适合多轮对话，也通常不作为定时任务执行对象。

![快捷输入模式](..\image\快捷输入模式-深色.gif)

### 💬 独立窗口 (Standalone Window)

* **场景**：适合多轮对话、复杂分析、持续上下文处理、MCP 工具调用、Skill 协同与定时任务执行。
* **交互**：调用后，会打开一个功能完整的桌面独立聊天窗口。
* **特点**：
  * 支持多轮上下文；
  * 支持 MCP 工具与 Skill 技能；
  * 支持文件 / 图片等多模态输入；
  * 支持自动保存历史对话；
  * 支持通过全局追问向已打开窗口追加消息；
  * 可作为定时任务执行对象；
  * 可与 Super-Agent 协作组成多 Agent 工作流。

## 2. 创建与配置指南

在主控台的 **快捷助手** 页面中，点击底部的 **“+ 新增快捷助手”** 即可开始配置。

### 2.1 基础设置

* **名称 (Name)**：助手的唯一标识，也是您在桌面端内识别它的主要名称。
* **图标 (Icon)**：
  * 支持上传本地图片或填写网络图片地址；
  * 支持桌面端内置头像编辑 / 裁剪能力。
* **类型 (Type)**：决定助手如何匹配输入内容。
  * **通用 (General)**：可直接召唤，也可匹配文本 / 图片 / 文件输入；
  * **文本 (Text)**：仅匹配文本输入，可配合正则过滤；
  * **图片 (Image)**：适合截图、剪贴板图片、拖入图片场景；
  * **文件 (Files)**：适合选中文件、拖拽文件、粘贴文件路径等场景。

### 2.2 模型与参数 (Model Parameters)

* **模型选择**：可为每个助手单独指定模型。
* **系统提示词 (Prompt)**：定义助手的角色、语气、输出格式与工作规则。
* **思考预算 (Reasoning Effort)**：对支持推理的模型可设置推理强度。
* **自定义温度**：控制回复随机性与稳定性。

### 2.3 窗口与外观 (Window & Appearance)

* **窗口尺寸**：自定义独立窗口的默认宽高。
* **背景设置**：可配置背景图、透明度与模糊效果。
* **交互行为**：
  * **窗口置顶**：保持窗口在最前；
  * **失焦关闭**：适合查词类 / 快速处理类助手；
  * **自动保存聊天**：让会话自动写入本地历史目录。

### 2.4 高级集成 (Integrations)

* **默认 MCP 服务**：指定该助手启动时自动挂载的工具集合。
* **默认 Skill 技能**：指定启动时默认启用的技能库。
* **语音设置**：给该助手指定默认语音输出音色或语音模型。

## 3. 调起方式与桌面端工作流

Anywhere Desktop 中，快捷助手的使用方式比原插件更偏向“系统级桌面体验”：

### 3.1 主控台直接打开

在主控台快捷助手页中直接点击对应助手，即可打开 Fast 或 Window 模式。

### 3.2 全局快捷键绑定

您可以在桌面端设置中，为某个快捷助手配置专属全局快捷键。之后无论当前焦点在哪个软件中，都可以直接召唤该助手。

### 3.3 Quick 快捷召唤窗口

通过桌面端的快捷召唤入口，您可以快速从多个快捷助手中选择一个，并将当前文本、图片或文件投递给它。

### 3.4 全局追问 (Append / Follow-up)

当某个独立窗口已打开时，您可以在系统任意位置复制文本、图片或文件，再通过桌面端全局追问能力，把这些素材继续发送到目标窗口。

这使得 Anywhere Desktop 特别适合“边工作边向 AI 追加上下文”的真实桌面办公场景。

---

## 4. 标签与管理

当助手数量增多后，可通过 **标签 (Tags)** 进行组织：

* **创建标签**：在快捷助手页中新建标签。
* **分配助手**：一个助手可属于多个标签。
* **批量操作**：支持按标签批量启停、批量替换模型等。

这样可以把编程、办公、写作、运维等不同用途的助手分组管理。

## 5. 常见问题 (FAQ)

**Q1: 什么是“全局追问”？如何使用？**

> **A:** 全局追问允许您在其他软件中复制文本、图片或文件后，将它们直接发送给已打开的 Anywhere Desktop 独立窗口。
>
> **常见用法：**
> 1. 先打开一个独立对话窗口；
> 2. 在任意软件中复制内容；
> 3. 通过桌面端全局追问快捷键或追加面板，把素材投递到目标窗口；
> 4. 若有多个窗口，可在追加面板中选择目标窗口。

**Q2: 为什么我召唤助手后没有出现？**

> **A:** 请检查：
>
> 1. 助手是否启用；
> 2. 助手类型是否限制了输入类型；
> 3. 是否绑定了错误的快捷键；
> 4. 当前调用的是 Fast 模式还是 Window 模式，是否被失焦关闭等行为立即收起。

**Q3: 为什么我创建的助手不能用于定时任务？**

> **A:** 定时任务通常要求目标助手具备完整上下文与工具调用能力，因此建议使用 **独立窗口 (Standalone Window)** 模式的助手。

**Q4: “文本直接发送”和“补充文本信息”有什么区别？**

> **A:**
>
> * **文本直接发送**：开启后，投递文本会直接发给 AI；关闭后，仅进入输入框草稿，由您手动确认。
> * **补充文本信息**：用于避免某些模型在纯图片 / 纯文件场景下因空文本而请求失败，系统会附带一条辅助说明文本。

**Q5: 正则匹配有什么用？**

> **A:** 它可用于让文本类快捷助手只在特定内容下出现。例如只匹配 B 站链接、GitHub 仓库地址、报错栈等特定格式文本。

**Q6: 助手配置丢失了怎么办？**

> **A:** 如果您已配置 WebDAV 备份，可从云端恢复配置。也建议定期导出桌面端配置文件留档。
